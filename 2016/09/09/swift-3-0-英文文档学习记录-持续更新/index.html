swift 3.0,英文文档学习记录(to self)(持续更新)

完全是swift3.0 beta来的,看一点写一点,未必理解到位,后续会持续更新和纠正(纠正过的会标记出来),就是一个学习的记录

  1. print 用于输出,等同于OC的NSLog,但是不用占位符,不用’@‘符,一个字+一个字,清爽,不同类型的输出用逗号隔开几即可,拼接的话只需要用’+’连接即可.
  2. 常量用let,变量用var,和oc的有些不同,;let修饰一些你不想变(想变也变不了)的量,哪怕是字符串什么的,而oc中,常量就指的一些类型,能不能变决定不了,比如int,oc中你随便变就是啦.总结来说:指向let声明的常量的’指针’(如果有这一说)是不能变的,而指向var声明的变量的指针是可以变的.至于这个常量或者变量的内部是否可变并不受现在,由其本身属性决定,比如:let声明的NSMutableDictionary字典是可以改变其中的值的
  3. ‘?’的作用,告诉编译器,这个量的值是可选的(optional),可以包含一个值也可以为nil,如果不带’?’,又不给值,系统就会报错,让你必须初始化
  4. if 条件语句的调只能是Boolean类型,但是像’let A = B’也可以作为条件,但是B需要时optional类型的,如果为B不为空,就把B赋值给A,并且条件为ture,反之,B为空的话,就条件就为fasle,而且A就没了,对,没了
  5. switch 语句的break关键字不需要了,但是仍然要有default
  6. forin语句,如果遍历的是字典,是可以同时遍历value和key的,而且系统在编译的时候就会判断,例如你不是字典类型的元素,非要按key-value方式来判断,它就会报错
  7. 类的声明中,属性必须要有一个初始值,无论是哪种方式,直接赋值,或者同过带参数init函数来初始化,反正要有,类的声明,可以是没有基类的,所以你在声明一个类时,可以不继承任何类.当然你在OC中也可以这样写,但是,但是,你在这个类就是什么方法都没有,生成器都没,但是swift中,你依然可以用()来创建,无论是父类还是子类,只要有属性没有直接赋予初始值,那么你在创建时都要传入缺少初始值的属性的值
  8. 属性声明也可以自定义setter和getter方法,但是在swift中,有setter和getter的属性,一般被视为计算属性,和属性本身其实关系不大,用OC的思想可能有点费解….例如:有个A属性,有个B属性,有个C属性,C属性有setter和getter方法,C可能就是功能性,比如A和B组合的一个效果什么的sum啥的
  9. 枚举,定义方式跟OC差不多,但是灵活的很多,使用范围更广,不限于整型,而且可以使用方法,用起来更贴近于自然的方式,贴近于人类的思考方式,而且每一个枚举的元素会有rawValue属性(只有在枚举声明的时候指明成员类型的才有,因为只有枚举本身有rawType的时候,枚举的成员才有rawValue),也就是你写在枚举名后面的值,不写的话会默认为枚举名,打印的时候也会以枚举名显示,OC中默认会显示rawValue,不论你写的什么GenderMan, GenderWoman,打印出来都是整型值.下面是随便写的一些枚举,是不是很灵活啊

还不是看起来像是一个类的定义,只不过把class关键字换成了enum,哈哈,如果是struct就更像了,官方的说法就是,他们确实有很多相似之处,但是最大的不同就是,结构体的使用都是拷贝方式,你引用几次就拷贝几次,但是class就不同,他是真的是引用,你可以理解为,同一个对象,你在不同的地方用,其实只是用不同的指针指向它而已,可以说,class更加节省空间,附上原话,免得说我瞎扯,😁

“Use struct to create a structure. Structures support many of the same behaviors as classes, including methods and initializers. One of the most important differences between structures and classes is that structures are always copied when they are passed around in your code, but classes are passed by reference.”
摘录来自: Apple Inc. “The Swift Programming Language (Swift 3)”。 iBooks.

  1. 上面也提到结构体,其实你可以认为swift中所有类型的对象都是结构体,只是说他们各自有一些特殊的属性,枚举也是一种结构体

下面就进入正式的部分,以上只是overview部分

  1. swift里面的数据类型:Int(整型),Float/Double(浮点型),Array/Set/Dictionary(集合),String(文本,字符串),Bool(布尔类型),tuples(元组,Objective-C中没有的)
  2. 使用let声明常量,var声明变量,和数据类型没有关系,不需要变的你就要let,变的你就用var.使用let 和 var的时候都是可以连续声明的,比如let a = 1, b = 2, c = 3这种.无论是var还是let在使用前必须要初始化,不然报错,在类型注释后面加?也是初始化的一种(但是只适用于变量,常量声明的时候哪怕你在类型注释后加了?,你依然要在试用前给他一个初始值),意思是你没指明初始值,那系统就会给你一个nil值.常量名和变量名可以用基本上所有的东西来命名,汉字,emoji等等,但是常见的一些禁用方式,依然是禁用的,什么数字开头,空格,私有或者保留的字段,-,等等
  3. swift的注释和 Objective-C类似,但是多行注释是可以嵌套多行注释的.
  4. 分号;在swift中不是必须的,你想写,也没人说你,但是如果在一行写多个语句必须用;隔开
  5. 整型(signed(+,0,-)/unsigned(0,+)),常用的就是Int,系统会自动根据系统的位数来判断取值范围,当然你也可以用Int8,Int32,Int64来声明变量或常量,但是,其实不常用,如果是无符号整型,用UInt,官方推荐用Int,无论是从安全还是从效率方面.
  6. 浮点型,signed浮点型包括Float(32位)和Double(64位)

  7. typealias,关键字,等同于type define,例如

  8. Bool类型只有两个值ture 和 fasle ,这个和非0即真还是有很大区别的

  9. Tuple,元组,swift中新增的类型,其实很像结构体,有没有

元组的元素,可以是任何类型的数据,元组在返回值是多个的时候特别有用

  1. Optionals,可选类型,这个就是之前(c或者Objective-c中)没有的,用处是,直接表现就是在类型的后面加上?,没有指明是可选类型的变量(对,变量,常量单独说),是不能被赋予nil值的,或者是,你使用这个变量的时候必须要手动初始化,但是加上这个?以后,你就可以不必非要手动初始化,系统会自动把它的值设为nil,还有对于未知累的结果也要带上?,防止未知的结果是空,这些注意事项,编译时都是会报错的,如果没有遵守.例如:

  2. nil,swift中的nil可以赋值给任何对象,不限于是不是基本数据类型,因为,它并不是一个指针,它就是一个值,特殊的值,通吃的值.Objective-c中,nil是指向空的一个指针,所以,它只能赋值给实例对象
