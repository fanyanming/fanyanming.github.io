php学习记录

语法

  1. 脚本开始和结尾:<?php ?>
  2. 声明变量用’$’开头,命名风格和OC相似, 变量的作用域分为全局和局部,在函数外声明的变量为全局变量, 函数内部声明的变量为局部变量, 与OC不同的是全局变量是不能直接在函数内部访问的, 除非在要访问的全局变量之前加上global关键字修饰. 顺便提到另外一个关键字’static’,它的作用就是能让局部变量不会在函数执行完毕后被删除.’特别指出的是$符号并不是声明的时候才使用,而是只要是变量,这个符号一直伴随’
  3. PHP的两种输出方法: 1>echo 2>print;前者可以输出多个字符串,后者只能输出一个字符串,前者没有任何返回值,后者始终返回 1;
  4. php的数据类型: 字符串、整数、浮点数、逻辑(true和false)、数组、对象(类比OC的话感觉就是字典)、NULL(注:var_dump() 会返回变量的数据类型和值),
  5. 字符串相关的函数:
    1>计算长度:strlen()
    2>计算摸个字符串的位置:strpos(),如果未找到返回false
  6. 函数:
    1>. 形式:function funtionName() {

}
2>. 参数,就放括号里面即可, 需要注意的是默认参数:<?php
function setHeight($minheight=50) {
echo “The height is : $minheight
“;
}

setHeight(350);
setHeight(); // 将使用默认值 50
setHeight(135);
setHeight(80);
?>
这里的例子就会在没有传参的时候用上50这个默认值
3>. 函数有返回值的时候,使用return即可,不用在函数声明的时候特意指出,OC的函数是需要的特别指明的

小程序上手记录

只记录,不论对错,错了就改,毕竟小白

  1. 小程序又三个文件构成,分别是app.js(程序逻辑,必需), app.json(程序公共设置,必需), app.wxxs(公共样式,非必需),这三个是必须要放在根目录下
  2. 小程序是又页面来展示的,而每个页面又是由四个文件组成, 分别以.js(页面逻辑, 必需), .wxml(页面结构, 必需), .wxss(页面样式, 非必需), .json(页面配置, 非必需)组成
  3. 两者的构成文件其实是一样的, 有人说页面的构成多一个, 或者说app的文件少了一个页面的结构, 那是因为app是有page构成, 它并不需要自己的页面结构.
  4. .js,放函数的, .wxml写UI的,.wxss定义约束的,.json,整个页面的一些属性,标题啊什么的.
  5. 所有你要展示的界面都要在 app.json中注册,这一点就跟你写oc要导入头文件一样,不然系统不知道这玩意是啥,在哪.
  6. 小程序中的导航栏是tab是在app.json中的设置的,位置是系统默认的,样式也是,你不需要单独来写,但是是可以修改这些样式,图片啊,背景啊,颜色啊,等等.定义的时候最少2个,最多5个.
  7. tabbar的每一个tab对应一个页面,写上路径即可
  8. 注册程序调用App()函数,同理注册页面用Page(),相关的函数,出事数据都是在对应的小括号里进行的,如果进行类比的,你可以理解成OC中的类方法的实现部分,就是.m文件,里面可以写属性,可以写方法,格式要简单一些而已.所以它也会有一些自己的生命周期的回调方法,你可以在各自的方法中监听,做相应的处理.getApp()方法可以在任意页面中调用,相当于OC中的单例,但是在有些方法中是不可以调用的(参考官方文档),类比的话就是避免循环引用,比如getApp()会调用someFunction()方法,你又在someFunction()中调用getApp(),那不就出事了的,是吧..
  9. 小程序的作用域,我感觉和OC是差不多的,.js中定义的方法,只能在相同的文件(夹)下使用,所以不同的文件下可以同名.

swift 3.0,英文文档学习记录(to self)(持续更新)

完全是swift3.0 beta来的,看一点写一点,未必理解到位,后续会持续更新和纠正(纠正过的会标记出来),就是一个学习的记录,并且是按着文档的先后顺序来写的

Welcome to Swift

  1. print 用于输出,等同于OC的NSLog,但是不用占位符,不用’@‘符,一个字+一个字,清爽,不同类型的输出用逗号隔开几即可,拼接的话只需要用’+’连接即可.
  2. 常量用let,变量用var,和oc的有些不同,;let修饰一些你不想变(想变也变不了)的量,哪怕是字符串什么的,而oc中,常量就指的一些类型,能不能变决定不了,比如int,oc中你随便变就是啦.总结来说:指向let声明的常量的’指针’(如果有这一说)是不能变的,而指向var声明的变量的指针是可以变的.至于这个常量或者变量的内部是否可变并不受现在,由其本身属性决定,比如:let声明的NSMutableDictionary字典是可以改变其中的值的(忽略本句,理解有混淆,不准确)
  3. ‘?’的作用,告诉编译器,这个量的值是可选的(optional),可以包含一个值也可以为nil,如果不带’?’,又不给值,系统就会报错,让你必须初始化
  4. if 条件语句的调只能是Boolean类型,但是像’let A = B’或者是多个let A = B,let B = C,...(一假则假)也可以作为条件,但是B需要时optional类型的,如果为B不为空,就把B赋值给A,并且条件为ture,反之,B为空的话,就条件就为fasle,而且A就没了,对,没了
